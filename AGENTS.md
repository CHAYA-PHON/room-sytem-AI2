# Sabaidee Dormitory - Developer & Versioning Rules

## 📌 Versioning Rule (กฎเกณฑ์การตั้งเลขเวอร์ชัน)
ทุกครั้งที่มีการแก้ไขแฟ้มข้อมูลหรือปรับปรุงระบบ ให้ทำการเปลี่ยนเลขเวอร์ชันที่แสดงผลในแถบแสดงผลของแอป (ไฟล์ `src/App.tsx`) ตามระเบียบดังนี้:
- รูปแบบหลัก: `V.YYYY-PBXX_M.B`
  - `YYYY` = ปีที่เผยแพร่ (เช่น `2026`)
  - `PBXX` = รอบที่เผยแพร่/Publish และเพิ่มรอบเมื่อมีคำสั่งส่งโค้ดไปยัง GitHub
  - `M` (เลขหลังขีดล่างตัวแรก) = เพิ่มขึ้นเมื่อมีการเพิ่มหรือขยายระบบใหม่เล็ก ๆ น้อย ๆ
  - `B` (เลขหลังจุดของ M) = เพิ่มขึ้นตามจำนวนครั้งของการแก้ปัญหาและแก้ไขบั๊ก (เช่น แก้ไขบั๊กครั้งแรกเป็น `_0.1`)

**เวอร์ชันปัจจุบันหลังการแก้ไขนี้:** `V.2026-PB02_1.6` (แจงรายละเอียดค่าเฉลี่ยแยกรายห้องพัก เพิ่มสถิติสูงสุด-ต่ำสุด และละเว้นค่าศูนย์หรือมิเตอร์เสียอัตโนมัติ)

## 📝 ประวัติย่อการอัปเดตรุ่น (Quick Version History)
*(ดูรายละเอียดการปรับปรุงแบบสมบูรณ์ที่ไฟล์ `CHANGELOG.md` ใน root ของโปรเจกต์)*
- **`V.2026-PB02_1.6`**: ปรับปรุงตารางแจงรายละเอียดเฉลี่ยรายห้องพัก เพิ่มการแสดงค่าสูงสุด-ต่ำสุดของแต่ละห้องพัก และข้ามรอบบิลที่หน่วยการใช้เป็น 0 หรือแจ้งมิเตอร์เสียอัตโนมัติ
- **`V.2026-PB02_1.5`**: ปรับแต่งและลดขนาดการ์ดแสดงสถิติเฉลี่ยรายเดือนสะสม 12 เดือน (กระชับและดึงสถิติน้ำ/ไฟเฉลี่ยขึ้นมาแสดงด้านข้างชื่อเดือน)
- **`V.2026-PB02_1.4`**: แก้ไขข้อผิดพลาดชื่อผู้เช่าไม่แสดงในการวิเคราะห์สถิติ, ปรับการคำนวณระยะเวลาเข้าพักตามรอบบิลจริง และปรับปรุงกลไกซ่อน/แสดงรายละเอียดค่าเฉลี่ยรายห้องพัก
- **`V.2026-PB02_1.3`**: กู้คืนและจัดระเบียบตารางแสดงรายละเอียดค่าเฉลี่ยรายห้องพักในแท็บ Seasonal Analysis, จัดยอดสะสมรุ่นให้ถูกต้อง
- **`V.2026-PB02_1.2`**: ถอดการ์ดเชื่อมต่อบัญชี Google ตรงออกจาก Admin Settings ตามผู้ใช้ขอ
- **`V.2026-PB02_1.1`**: ปรับแต่ง Responsive Layout และ Grid ของแท็บข้อมูลเฉลี่ยในหน้า AI Analytics
- **`V.2026-PB02_1.0`**: เปิดตัวฟีเจอร์ AI Analytics เต็มรูปแบบ (การวิเคราะห์รายฤดูกาล, เปรียบเทียบ และสรุปข้อมูลการใช้พลังงาน)
- **`V.2026-PB02_0.2`**: ออกแบบการ์ดประวัติห้องพักให้กะทัดรัด (Compact Design)
- **`V.2026-PB02_0.1`**: แก้ไขปัญหาทศนิยมและการคำนวณเบื้องต้นของค่าสถิติเฉลี่ย
- **`V.2026-PB01_1.0`**: เผยแพร่แอปเวอร์ชันแรกอย่างเป็นทางการ พร้อมระบบ Auto-Pull จาก Google Sheets

## 💡 คู่มือแนะนำการใช้งานระบบนิติบุคคล (System Usage Instructions)
- ห้ามใช้คำอื่น ให้ใช้คำว่า **"แนะนำการใช้งาน"** สำหรับส่วนของคำชี้แจงผู้ใช้งาน
- เมื่อเปิดตัวแอปพลิเคชันขึ้นมา ให้เปิดใช้งานระบบดึงข้อมูลอัตโนมัติ (Auto-Pull) จาก Google Sheets ทันทีในพื้นหลัง โดยไม่ต้องรอให้ผู้ใช้นิติบุคคลล็อกอินเข้าระบบ เพื่อรักษาข้อมูลที่สดใหม่และอัปเดตล่าสุดอยู่เสมอ
